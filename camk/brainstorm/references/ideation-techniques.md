# Ideation Techniques

Methods for generating diverse, creative ideas.

## SCAMPER

Prompts for rethinking existing solutions:

| Letter | Question | Example |
|--------|----------|---------|
| S | Substitute | What if we replace X with Y? |
| C | Combine | Can we merge two approaches? |
| A | Adapt | How do others solve this? |
| M | Modify | What if we change the scale? |
| P | Put to other use | Where else could this work? |
| E | Eliminate | What if we remove this step? |
| R | Reverse | What if we do the opposite? |

## Six Thinking Hats

| Hat | Perspective | Question |
|-----|-------------|----------|
| White | Facts | What data do we have? |
| Red | Emotions | How do users feel? |
| Black | Caution | What could go wrong? |
| Yellow | Optimism | What are the benefits? |
| Green | Creativity | What new ideas emerge? |
| Blue | Process | Are we on track? |

## First Principles

1. Break down the problem to fundamental truths
2. Question every assumption
3. Build up from basics
4. Avoid reasoning by analogy

Example:
```
Assumption: "We need a database"
First principle: "We need to store and retrieve data"
Question: "Can we use files? Cache? External service?"
```

## Constraint-Based

Deliberately add constraints to force creativity:
- "What if we had half the budget?"
- "What if we had one week?"
- "What if we couldn't use X technology?"
- "What if we had 10x the users?"
