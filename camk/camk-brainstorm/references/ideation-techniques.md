# Ideation Techniques

Methods for generating diverse, creative ideas.

---

## SCAMPER

Prompts for rethinking existing solutions:

| Letter | Question | Example |
|--------|----------|---------|
| S | Substitute | What if we replace X with Y? |
| C | Combine | Can we merge two approaches? |
| A | Adapt | How do others solve this? |
| M | Modify | What if we change the scale? |
| P | Put to other use | Where else could this work? |
| E | Eliminate | What if we remove this step? |
| R | Reverse | What if we do the opposite? |

**When to use:** Bạn đã có 1 solution, muốn cải tiến nó.

---

## Six Thinking Hats

| Hat | Perspective | Question |
|-----|-------------|----------|
| White | Facts | What data do we have? |
| Red | Emotions | How do users feel? |
| Black | Caution | What could go wrong? |
| Yellow | Optimism | What are the benefits? |
| Green | Creativity | What new ideas emerge? |
| Blue | Process | Are we on track? |

**When to use:** Cần nhìn problem từ nhiều góc độ. Đặc biệt hữu ích khi team bị stuck ở 1 perspective.

---

## First Principles

1. Break down the problem to fundamental truths
2. Question every assumption
3. Build up from basics
4. Avoid reasoning by analogy

**Example:**
```
Assumption: "We need a database"
First principle: "We need to store and retrieve data"
Question: "Can we use files? Cache? External service?"
```

**When to use:** Problem phức tạp, solutions truyền thống không hiệu quả.

---

## Constraint-Based

Deliberately add constraints to force creativity:
- "What if we had half the budget?"
- "What if we had one week?"
- "What if we couldn't use X technology?"
- "What if we had 10x the users?"
- "What if we had to build this with 1 engineer?"

**When to use:** Ideas đang quá safe, cần force creativity.

---

## Analogies

"Nếu đây là [domain khác], ta sẽ làm gì?"

| Domain | Analogy | Insight |
|--------|---------|---------|
| Nature | "Cây cối phân phối nước như thế nào?" | Distributed systems |
| Sports | "F1 pit stop tối ưu như thế nào?" | Parallel processing |
| Cooking | "Michelin kitchen organize workflow ra sao?" | Team coordination |
| Military | "Special forces plan missions?" | Risk assessment |

**When to use:** Cần fresh perspective, domain hiện tại bị stuck.

---

## Random Word Association

1. Pick a random word (from dictionary, book, or generator)
2. Force connection between that word and your problem
3. See what ideas emerge

**Example:**
```
Problem: Improve user onboarding
Random word: "Bridge"
→ "Bridge the gap between signup and first value"
→ "Onboarding as a bridge, not a tunnel"
→ "What if onboarding has toll gates (milestones)?"
```

**When to use:** Brainstorm bị stuck, cần jolt creativity.

---

## Reverse Brainstorming

Instead of "How do we solve X?", ask "How do we CAUSE X?"

1. Define the problem (e.g., "Users churn")
2. Ask: "How do we make users churn MORE?"
3. List all ways to cause the problem
4. Flip each into a solution

**Example:**
```
How to cause churn:
- Make onboarding confusing → Fix: Simplify onboarding
- Don't respond to support → Fix: Fast support SLA
- Hide key features → Fix: Feature discovery prompts
```

**When to use:** Problem đã rõ nhưng solutions không hiệu quả.

---

## Worst Possible Idea

1. Deliberately generate the WORST ideas
2. Make them as terrible as possible
3. Then ask: "What makes this bad?"
4. Flip the bad elements into good solutions

**When to use:** Team quá nghiêm túc, cần lower the barrier.

---

## How to Choose a Technique

| Situation | Technique |
|-----------|-----------|
| Có solution cũ, muốn cải tiến | SCAMPER |
| Team bị stuck ở 1 góc nhìn | Six Thinking Hats |
| Problem phức tạp, cần breakthrough | First Principles |
| Ideas quá safe, cần radical | Constraint-Based |
| Cần fresh perspective | Analogies |
| Brainstorm bị stuck | Random Word / Worst Idea |
| Problem rõ nhưng solutions yếu | Reverse Brainstorming |
